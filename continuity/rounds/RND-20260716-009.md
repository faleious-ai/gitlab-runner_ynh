## Design outcomes

Preserved from MAESTRO:

- layered repository memory;
- orchestrator/executor separation;
- independent external review;
- autonomous reversible technical decisions;
- proportional human gates;
- DAG/subagents with no commit authority;
- direct `master`, no PR/branch/worktree;
- remote persistence as delivery.

Imported/adapted from Cavekit:

- verification contract per claim;
- seam-driven TDD;
- adversarial spec challenge;
- two-axis pre-commit review;
- backpropagation and pattern escalation;
- drift checking and evidence levels;
- dead-end memory;
- convergence/ceiling distinction;
- bounded token compression;
- task commits.

Rejected:

- one-file spec;
- no subagents/no parallelism;
- Claude-specific runtime/hooks;
- branch/agent commits;
- human approval for reversible technical backprop;
- grep or diff size as sufficient proof.
